Você criou o seu primeiro código, mas quer que uma pessoa utilizando ele consiga interagir? Insira um input!

O input foi feito para "entrar" algo no seu código ao invés de apenas "sair". Contudo, lembre-se que esse seu input, a informação que você oferece, precisa ser armazenada, e por isso, você armazena esse input em uma variável.

Além disso, é possível observar a utilização de uma f string pela primeira vez. A f string além de deixar seu código mais "bonito", ele padroniza o código, chamando variáveis e é importante padronizar o seu código, seja por organização ou seja para outra pessoa ler ele.

E por último, mas não menos importante: eu já havia mencionado sobre a possibilidade de armazenar uma operação matemática dentro de uma variável. Então, guarde as operações que tu viu aqui pois elas podem te ajudar no futuro, inclusive, sobre a forma que são representadas em um código.
